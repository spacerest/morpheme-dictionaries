You are creating morpheme dictionary entries for a language-learning word puzzle game. The player knows <HOMELANG> and is learning <TARGETLANG>. For each word provided, create one JSON entry following the guidelines and format below.

**Entry guidelines:**

1. **id**: Use the <TARGETLANG> word exactly as given (lowercase unless language-specific requirements, like uppercase nouns in German, requires it, otherwise no changes).
2. **article**: The definite article for the word if applicable (e.g., "der", "die", "das" for German nouns; "le", "la" for French), or "" for verbs, uninflected forms, and languages without gendered articles. For languages like Russian, store the grammatical gender marker in parentheses as in (м), (ж), or (с))
3. **displayPrefix**: An optional particle displayed before the article and word in the UI but not part of gameplay (e.g., "sich" for German reflexive verbs, "se" for French). Omit if not applicable.
4. **parts**: Split the word at all real morpheme boundaries, even when a component is itself a recognizable word. For example, "verabschieden" should be ver + ab + schied + en, not ver + abschied, because "abschied" is itself a compound. Aim for 2–4 parts; if full decomposition yields 5 or more, include them all and add a flag. Include linking morphemes (e.g., German -s-) as their own part with "homeLang": "-". Grammatical affixes get a brief label like "(infinitive)" or "(plural)".

   **CRITICAL — surface forms:** The `targetLang` values for all parts must concatenate exactly to the word's `id` (after stripping hyphens used as prefix/suffix markers). Always use the surface form as it appears in the word, never the underlying etymological form. Common English spelling mutations to watch for:
   - Roots change before certain suffixes: `scrib` → `scrip` in *prescription* (`pre`+`scrip`+`tion`), `vert` → `vers` in *conversion* (`con`+`vers`+`ion`)
   - Final `-e` is dropped before vowel suffixes: *believable* = `believ`+`able`, *reversible* = `revers`+`ible`, *recognizable* = `recogniz`+`able`
   - `-ize`+`-ation` → `-iz`+`-ation`: *organization* = `organ`+`iz`+`ation`
   - `-ify`+`-ation` → `-ific`+`-ation`: *simplification* = `simpl`+`ific`+`ation`
   - `-ate` drops the `e` before `-or`/`-ion`: *investigator* = `invest`+`igat`+`or`, *communication* = `commun`+`icat`+`ion`
   - Connecting vowels must be included as parts: *chronology* = `chron`+`o`+`logy`, *geography* = `geo`+`graph`+`y`
   - For `-ive` words: use `ive` before `-ly` (*effectively* = `effect`+`ive`+`ly`) but `iv` before `-ity` (*creativity* = `creat`+`iv`+`ity`)
   - For `-ment` words where the stem ends in `-ve`: use `-ement` (*involvement* = `in`+`volv`+`ement`)
   - Double consonants belong to one part: *commitment* = `com`+`mit`+`ment` (not `mitt`)
5. **homeLang values**: Each part's homeLang is a building block the player uses to puzzle out the word's meaning — do not pre-solve the puzzle for them by combining parts in advance. Each tile must reflect only what that morpheme contributes on its own. It must be SHORT — 1 word ideally, 2 at most. Rules: (a) Gloss the morpheme itself, not the compound it appears in. `druck` in `Ausdruck` means `press`, not `expression` — `expression` is what `aus` + `druck` mean together, and the player should arrive at that themselves. Other examples of this mistake: glossing `fahr` in `Erfahrung` as "experience" instead of "travel/go", or `stand` in `Verstand` as "understanding" instead of "stand". (b) ONE meaning only — no slashes. Put alternatives in homeLangDetails. (c) Where a suffix or prefix has a direct English equivalent, use it: -heit → "-ness", -los → "-less", -bar → "-able", -lich → "-like". (d) Context-dependent prefixes get a plain English word — no parentheses: ver- might be "away" or "astray" depending on the word. (e) Grammatical inflections use dash notation if there's an English equivalent (-en plural → "-s", ge- past participle → "-ed"), or a parenthetical label if not (-en infinitive → "(verb)"). Dash-notation labels like "-ing", "-ness", "-less", "-es" (or their equivalents in other home languages) are semantic building blocks — they show the player what the morpheme contributes and should be used whenever a direct home-language suffix equivalent exists. Reserve parenthetical labels like "(infinitive)" or "(plural)" only for inflections that have no clean home-language suffix equivalent. <CANONICAL_LABELS>
6. **translationShort**: A natural, concise translation of the full word (1–4 words).
7. **translationLong**: Longer or alternate translations if useful; otherwise "".
8. **homeLangDetails**: For each part, ask yourself: does the glossary entry for this morpheme describe what it's doing in *this specific word*? If yes, omit homeLangDetails entirely — the glossary text will be auto-filled. If no — or if there is no glossary entry — write one. Include it in these situations: (a) the morpheme behaves context-specifically here, beyond its standard glossary meaning (e.g., ver- means something different in versprechen vs verabschieden); (b) the part's text resembles a glossary morpheme but is NOT that morpheme (e.g., the letters "-er" are part of the root, not the agent suffix); (c) there is no glossary entry for this morpheme. When in doubt, include it — a wrong or misleading auto-fill is worse than a custom note.

   **Style when writing homeLangDetails:** 2-3 sentences, warm and conversational — like a knowledgeable friend, but precise enough for a dictionary. Never open with "From X (gloss)..." — write "As in schreiben (to write)..." instead. The morpheme is the root; the full verb or noun is built on it, not the other way around. This is a hard rule, no exceptions. Lead with something that genuinely adds context: a cognate, a related word, or what makes this morpheme's behavior interesting here. Connect to an English cognate if one exists (just use it, don't explain what a cognate is). End with 1-2 other words in the target language that share the root. Use commas instead of em-dashes. Do not add meta-commentary ("isn't it interesting that..."), do not restate what was already said in other fields.
9. **exampleSentence**: A natural, grammatically correct sentence in <TARGETLANG> using the word.
10. **exampleTranslation**: Accurate <HOMELANG> translation of the example sentence.

If a word has only 1 morpheme part or 5+ parts, add a "flag" field with a brief note (e.g., "flag": "only 1 part"). Otherwise omit "flag".

Return only valid JSON — no explanation, no markdown code fences. Output a single object with a "words" array containing one entry per word. Use standard ASCII punctuation for all JSON structure (commas, colons, quotes, brackets) — never fullwidth or locale variants (，：""「」 etc.), even when the content language uses them. Never use ASCII double quotes (") inside Chinese or Japanese string values — use 「」for quotation instead (e.g. 表示「进入」not 表示"进入").
