You are creating morpheme dictionary entries for a language-learning word puzzle game. The player knows <HOMELANG> and is learning <TARGETLANG>. For each word provided, create one JSON entry following the guidelines and format below.

**Entry guidelines:**

1. **id**: Use the <TARGETLANG> word exactly as given (no changes).
2. **article**: Store the grammatical gender marker in parentheses: (м) masculine, (ж) feminine, (с) neuter. Use "" for verbs and indeclinable words.
3. **displayPrefix**: Omit unless applicable (not commonly needed for Russian).
4. **parts**: Split the word at all real morpheme boundaries. **CRITICAL for Russian verbs:** A prefixed verb must always be split — never treat the prefixed form as a single morpheme. Common patterns:

   - **Prefixed verbs**: prefix + root + infinitive ending. сделать = с + дел + ать (NOT: сделать as one part, NOT: с + делать). написать = на + пис + ать. выйти = вы + йт + и.
   - **Imperfective/perfective pairs**: the prefix changes the aspect, so it must be a separate part. делать (imperfective) = дел + ать. сделать (perfective) = с + дел + ать.
   - **Infinitive ending**: -ть or -ти is always its own part with homeLang "(verb)".
   - **Noun suffixes**: -ость, -ение/-ание, -ник, -ка etc. are always their own parts.
   - **Adjective endings**: -ный/-ная/-ное etc. are always their own parts.

   Always use the surface form as it appears in the word. The `targetLang` values for all parts must concatenate exactly to the word's `id`.

5. **homeLang values**: Each part's homeLang is a building block the player uses to puzzle out the word's meaning — do not pre-solve the puzzle. Each tile must reflect only what that morpheme contributes on its own. SHORT — 1 word ideally, 2 at most. Rules: (a) Gloss the morpheme itself, not the compound. дел in сделать means "do/make", not "complete" — the с- prefix contributes the completion sense. (b) ONE meaning only — no slashes. (c) Aspectual prefixes get their most concrete English gloss: с- here means "together/down" but in сделать it's a pure perfectivizer — use "done-" or "compl." only if nothing more concrete fits; otherwise use the spatial/directional meaning from the glossary. (d) Grammatical endings use parenthetical labels: -ть/-ти → "(verb)", -л- past → "(past)", noun endings → "(noun)" etc. <CANONICAL_LABELS>
6. **translationShort**: A natural, concise translation of the full word (1–4 words). For verbs, include "to": "to make", "to write".
7. **translationLong**: Longer or alternate translations if useful; otherwise "".
8. **homeLangDetails**: Write one when: (a) the morpheme behaves context-specifically beyond its standard glossary meaning; (b) there is no glossary entry for this morpheme; (c) the letters resemble a known morpheme but are NOT that morpheme. Omit if the glossary entry describes this use accurately — it will be auto-filled.

   **Style:** 2–3 sentences, warm and conversational. Never open with "From X (gloss)...". Lead with a cognate, related word, or what makes this morpheme interesting here. End with 1–2 other Russian words sharing the root. Use commas, not em-dashes.
9. **exampleSentence**: A natural, grammatically correct Russian sentence using the word.
10. **exampleTranslation**: Accurate English translation of the example sentence.

If a word has only 1 morpheme part or 5+ parts, add a "flag" field with a brief note. Otherwise omit "flag".

Return only valid JSON — no explanation, no markdown code fences. Output a single object with a "words" array containing one entry per word.
