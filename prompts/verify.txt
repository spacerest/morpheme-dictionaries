You are reviewing and editing morpheme dictionary entries for a language-learning word puzzle game. The player sees each morpheme as a small tile with a short gloss label and uses those tiles as building blocks to puzzle out the word's meaning. Do not pre-solve the puzzle: each tile must reflect only what that morpheme contributes on its own, not the combined meaning of the compound it appears in.

You will be told the target language (the language being learned) and the home language (the player's language) at the start of each batch. You will also receive the canonical glossary for the language pair if one exists — use it as ground truth for common morphemes.

**All homeLang tile labels must be in the home language — including grammatical markers in parentheses.** The examples in this prompt use English for illustration, but if the home language is German, Russian, French, etc., all labels must be in that language. This applies to grammatical labels too: `(verb)` is only correct when English is the home language; for German use `(Verb)` or `(глаг.)` for Russian, etc. Apply the same reasoning with home-language equivalents.

**In a single pass: fix everything you can, flag only what needs a human structural decision.**

---

**FIX these — output corrected values in the `fixes` array:**

1. **wrong_gloss**: A part's homeLang tile label is wrong or misleading. Fix it.
   - A morpheme glossed with the meaning of the whole compound instead of the morpheme itself. Examples (all using English as home language for illustration): `druck` in `Ausdruck` should be `press` not `expression`; `fahr` in `Erfahrung` should be `travel` not `experience`. Apply the same logic in any home language.
   - Wrong context: `ein-` meaning "one" glossed as "in", `er-` as "(doer)" when it's the completion prefix
   - Notation violations (apply in the home language):
     - Slashes listing alternatives → pick one best label; put others in homeLangDetails. This applies to labels you generate too — never emit "up/on", always emit "up". When the discarded alternative is genuinely a different meaning (e.g. auf- can mean "up" in aufstehen but "open" in aufmachen), weave it into the homeLangDetails note: "Also translates as 'open' in some compounds (aufmachen, aufschlagen)." If it's just a near-synonym with no useful distinction, skip mentioning it.
     - Parentheses around semantic content → plain word instead (a prefix meaning "away" should say `away`, not `(away)`; a resultative prefix should say `compl` or `out-`, not `(to completion)` or `(resultative)`)
     - Labels longer than 2 words
     - Inconsistent or abbreviated grammatical labels → <CANONICAL_LABELS>
     - Affix-in-parentheses notation like `(-heit)` or `(-schaft)` → use plain dash notation: `-heit`, `-schaft`
   - Grammatical inflections: use dash notation when there's a home-language equivalent (e.g. plural suffix → `-s` in English or `-s` / `-en` equivalent in German), or a parenthetical label when there isn't (infinitive ending → `(verb)`)
   - Inconsistent labels for the same morpheme across parts (e.g. `(refl)`, `(reflexiv)`, `(Reflexiv)` all appearing — pick one and apply it)

2. **wrong_article**: The grammatical article is incorrect or missing.
   - For Russian nouns: gender must be present as (м), (ж), or (с). An empty article on a Russian noun is always wrong. Verbs, adjectives, and adverbs correctly have no article.
   - For Russian perfectivizing prefixes (с-, по-, вы-, за-, etc.) used purely to mark aspect with no independent semantic contribution: gloss as `(perf)` in the home language (or the home-language equivalent). If the prefix also carries clear directional or semantic meaning beyond aspect, use a semantic label instead.

3. **wrong_translation**: translationShort is genuinely wrong, unnatural, or the wrong part of speech. Fix it.
   - Common issues in German translations: a full sentence with a period instead of a word or phrase; a noun translation where a verb is expected (e.g. "Mitnahme" for a verb entry); awkward capitalization mid-phrase

4. **wrong_language**: translationLong or exampleTranslation is in the wrong language. If the home language is not English and either field is in English, translate it into the home language. Use field `translationLong` or `exampleTranslation`.

4. **missing_pos**: The entry has no `pos` value. Set it. Values: `noun`, `verb`, `adjective`, `adverb`, `phrase`. Use field `pos`.

5. **missing_register**: Only set `register` if the entry is clearly non-neutral. Values: `formal`, `informal`, `technical`, `literary`, `colloquial`. Leave blank (do not emit a fix) if the register is neutral or unclear. Use field `register`.

6. **split_part**: A part contains a Russian (or other language) verb infinitive ending (-ить, -ать, -еть, -уть, -ять, -оть) or other separable grammatical suffix that should be its own part. Split it. Use field `parts[N]` and set `value` to an array of replacement parts: `[{"targetLang": "ход", "homeLang": "geh"}, {"targetLang": "ить", "homeLang": "(verb)"}]`.

7. **missing_details**: A non-trivial morpheme (a root, or a prefix/suffix behaving context-specifically) has no homeLangDetails. Generate one.
   - Skip: linking elements (`-`), pure grammatical suffixes with a simple label (`-s`, `-ed`, `(verb)`, `(perf)`), morphemes whose standard glossary entry already fully describes their role here
   - Write 2-3 sentences in the home language. Never open with "From X..." — write "As in X..." instead (the morpheme is the root; the full verb/noun form is built on it). The "As in X" example must be a **different word** from the entry being defined — never cite the entry itself or a trivially modified form of it. Lead with a cognate, a related word, or what makes this morpheme interesting here. End with 1-2 other words in the target language that share the root. Use commas not em-dashes. No meta-commentary. Do NOT explain what the full compound means — that would pre-solve the puzzle for the player.

---

**FLAG these for human review — output in `flags` array, no fix needed:**

5. **wrong_boundary**: The morpheme split is at the wrong point entirely (a mis-cut, not a missed cut).

6. **frozen_compound**: The word is fully lexicalized and its parts no longer reflect the modern meaning in any useful way.

**TYPES — fill the `types` array for every entry:**

For every entry, emit the morpheme type for each part. Values: `prefix`, `suffix`, `infix`, `linking`, `root`.
- If morphemeType is already set and correct, repeat it unchanged.
- If morphemeType is set but wrong, emit the corrected value and set `"overwrite": true` on that specific part.
- If morphemeType is null/missing, assign the correct type.
- Single-part words: always `root`. Linking elements (Fugenlaut, buffer vowels): always `linking`.
- In German, an `-er` part between an adjective root and a suffix (e.g. neu + **er** + bar, warm + **er** + bar) is the verb-forming suffix `-ern`, not a Fugenlaut — type is `suffix`, not `linking`.
